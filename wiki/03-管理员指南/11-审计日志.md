# 审计日志

本文档介绍审计日志的查看、查询和导出操作。

---

## 一、日志概述

### 1.1 日志作用

审计日志记录用户在系统中的所有操作，用于：
- 操作追溯
- 安全审计
- 问题排查
- 合规检查

### 1.2 日志入口

**路径**：系统管理 → 审计日志

---

## 二、日志查询

### 2.1 筛选条件

| 筛选项 | 说明 |
|--------|------|
| 用户名 | 操作用户 |
| 请求方法 | GET/POST/PUT/DELETE |
| 请求路径 | API路径 |
| 状态码 | HTTP状态码 |
| 操作时间 | 时间范围 |
| IP地址 | 操作来源IP |

### 2.2 日志列表

| 字段 | 说明 |
|------|------|
| 操作时间 | 操作发生时间 |
| 用户名 | 操作用户 |
| 请求方法 | HTTP方法 |
| 请求路径 | API路径 |
| 请求体 | 请求数据 |
| IP地址 | 来源IP |
| 状态码 | HTTP状态码 |

---

## 三、日志详情

### 3.1 查看详情

点击日志记录可查看详细信息：
- 完整的请求路径
- 请求体内容
- 响应状态
- 操作时间

### 3.2 敏感信息处理

日志中的敏感信息（如密码）会被脱敏处理，显示为 `******`。

---

## 四、日志导出

### 4.1 导出操作

1. 设置筛选条件
2. 点击「导出」按钮
3. 选择导出格式（Excel）
4. 下载文件

### 4.2 导出内容

导出的日志包含：
- 操作时间
- 用户名
- 操作内容
- IP地址
- 状态码

---

## 五、日志分析

### 5.1 操作统计

可查看以下统计数据：
- 操作次数统计
- 用户活跃度
- 接口调用频率
- 错误率统计

### 5.2 异常检测

系统自动检测异常操作：
- 频繁失败登录
- 异常访问路径
- 大量数据导出
- 权限越权尝试

---

## 六、日志保留

### 6.1 保留期限

| 日志类型 | 保留期限 |
|----------|----------|
| 登录日志 | 1年 |
| 操作日志 | 1年 |
| 审计日志 | 3年 |

### 6.2 日志归档

超过保留期限的日志自动归档，归档后可恢复。

---

## 七、常见问题

### Q1: 如何查看某个用户的所有操作？

**方法**：在筛选条件中输入用户名，点击查询。

### Q2: 日志可以删除吗？

**答案**：不可以。审计日志不可删除，确保审计完整性。

### Q3: 如何排查操作问题？

**方法**：
1. 根据时间范围筛选
2. 查看相关操作的请求和响应
3. 分析错误原因

### Q4: 日志数据量太大怎么办？

**方法**：
1. 设置合理的保留期限
2. 定期归档历史日志
3. 使用时间范围筛选查询

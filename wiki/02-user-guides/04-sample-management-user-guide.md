# 样品管理用户指南

## 适用对象

- 收样与样品管理员：登记、查询、处置样品。
- 检测调度：按样品维度跳转任务列表。

## 前置条件

- 已存在 **委托单**（`/entrustment`），且业务上允许收样。
- 权限：`sample:list`、`sample:create`（登记）；详情依赖列表权限。

## 页面入口

| 功能 | 路由 | 侧栏 |
|------|------|------|
| 样品列表 | `/sample` | 业务管理 → 样品管理 |
| 样品登记 | `/sample/register` | 列表「登记」或直达 |
| 样品详情 | `/sample/:id` | 列表点击进入 |

## 字段说明

### 样品（Sample）

| 字段 | 含义 |
|------|------|
| `sample_no` | 样品编号（唯一） |
| `blind_no` | 盲样编号（唯一，可空） |
| `commission` | 所属委托单 |
| `group` | 所属样品组（可选） |
| `name` | 样品名称 |
| `specification` | 规格型号 |
| `grade` | 设计强度/等级 |
| `quantity` / `unit` | 数量与单位 |
| `sampling_date` | 取样日期 |
| `received_date` | 收样日期 |
| `production_date` | 生产/成型日期（可选） |
| `sampling_location` | 取样地点 |
| `status` | 待检/检测中/已检/留样/已处置/已退还 |
| `retention_deadline` | 留样到期日 |
| `disposal_date` / `disposal_method` | 处置日期与方式 |
| `remark` | 备注 |

### 样品组（SampleGroup）

| 字段 | 含义 |
|------|------|
| `group_no` | 组样编号 |
| `name` | 组样名称 |
| `sample_count` | 组内样品数 |

### 样品处置（SampleDisposal）

记录退还、销毁、丢弃等及处置人、日期。

## 标准操作步骤（SOP）

### 登记新样品

1. 进入 **`/sample/register`**。
2. 选择 **委托单**（可结合委托编号搜索）。
3. 填写 **样品名称**、规格、等级、数量、取样/收样日期、取样地点等。
4. 如需盲样管理，填写或确认 **盲样编号** 规则（以界面为准）。
5. 保存后记录 **样品编号**，用于贴签与交接。

### 查询与详情

1. 在 **`/sample`** 按委托、日期、状态等筛选。
2. 进入 **`/sample/:id`** 查看完整信息、关联检测任务列表。

### 生成检测任务

1. 在样品详情按实验室流程点击 **「生成检测任务」**（若界面提供）。
2. 任务出现在 **`/testing/tasks`**；若列表为空，页面可能提示先完成生成（与 `TaskList` 中无任务提示一致）。

### 留样与处置

1. 对需留样的样品维护 **留样到期日** 与 **`status`** 为留样。
2. 处置时记录 **处置方式**、日期，保持与实验室管理规定一致。

## 常见错误

| 现象 | 原因 | 处理 |
|------|------|------|
| 检测任务列表为空 | 未生成任务 | 在样品详情生成任务；检查质量参数是否配置 |
| 委托选不到 | 委托已取消或权限不足 | 核对委托 `status` 与账号权限 |
| 盲样编号冲突 | `blind_no` 唯一约束 | 更换编号 |
| 从任务跳转样品失败 | ID 错误 | 使用 `/sample/:id` 正确主键 |

## 数据核对清单

- [ ] `commission` 与委托单一致；`sample_no` 与实物标签一致。
- [ ] 取样日期、收样日期符合逻辑顺序。
- [ ] 样品 `status` 与当前检测进度一致。
- [ ] 关联 **`/testing/tasks`** 中任务数量与委托项目行、拆分规则一致。

## 与上下游模块关系

- **上游**：**委托管理** `/entrustment`（必选委托）；可选 **项目** 信息经委托带出。
- **下游**：
  - **检测任务** `/testing/tasks`、`/testing/tasks/:id`：`TestTask.sample`。
  - **原始记录** `/testing/records`：通过任务间接关联样品。
  - **报告** `/reports`：委托维度汇总，样品为过程追溯单元。
- **跳转**：任务列表支持 `?sample=<id>` 查询参数筛选某样品任务（见检测任务页 `onMounted` 逻辑）。

相关路由：**`/sample`**、**`/sample/register`**、**`/sample/:id`**。

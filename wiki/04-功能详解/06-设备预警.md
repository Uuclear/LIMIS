# 设备预警

本文档详细介绍设备校准到期预警功能的实现和使用方法。

---

## 一、功能概述

设备预警功能用于监控实验室设备的校准状态，确保设备在有效期内使用，保障检测数据的准确性和可靠性。

### 1.1 预警目的

| 目的 | 说明 |
|------|------|
| 确保数据准确 | 使用校准有效期内的设备进行检测 |
| 合规要求 | 满足ISO/IEC 17025等标准要求 |
| 风险控制 | 避免使用过期设备导致的数据风险 |
| 计划管理 | 提前安排设备校准计划 |

### 1.2 适用范围

- 所有需要定期校准的检测设备
- 所有需要期间核查的辅助设备
- 所有需要维护保养的仪器设备

## 二、预警阈值设置

### 2.1 预警级别

系统设置两级预警机制：

| 预警级别 | 触发条件 | 颜色标识 | 说明 |
|----------|----------|----------|------|
| 正常 | 剩余天数 > 30天 | 绿色 | 设备校准状态正常 |
| 预警 | 剩余天数 ≤ 30天 | 黄色 | 提醒安排校准计划 |
| 紧急 | 剩余天数 ≤ 7天 | 橙色 | 紧急提醒，需立即处理 |
| 过期 | 已超过校准有效期 | 红色 | 设备不可使用 |

### 2.2 预警阈值配置

**配置路径**：系统设置 → 预警配置 → 设备预警

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 预警提前天数 | 30天 | 提前多少天开始预警 |
| 紧急提前天数 | 7天 | 提前多少天进入紧急状态 |
| 通知频率 | 每天 | 预警通知发送频率 |
| 通知时间 | 08:00 | 每天发送通知的时间 |

### 2.3 阈值计算逻辑

```python
def calculate_calibration_status(calibration_expiry_date: date, current_date: date) -> dict:
    """
    计算设备校准状态
    
    Args:
        calibration_expiry_date: 校准有效期截止日期
        current_date: 当前日期
    
    Returns:
        dict: 包含状态、剩余天数、预警级别等信息
    """
    days_remaining = (calibration_expiry_date - current_date).days
    
    if days_remaining < 0:
        return {
            'status': 'expired',
            'level': 'danger',
            'days_remaining': days_remaining,
            'message': f'设备已过期{-days_remaining}天，请立即校准'
        }
    elif days_remaining <= 7:
        return {
            'status': 'urgent',
            'level': 'warning',
            'days_remaining': days_remaining,
            'message': f'校准将在{days_remaining}天后到期，请紧急安排校准'
        }
    elif days_remaining <= 30:
        return {
            'status': 'warning',
            'level': 'info',
            'days_remaining': days_remaining,
            'message': f'校准将在{days_remaining}天后到期，请安排校准计划'
        }
    else:
        return {
            'status': 'normal',
            'level': 'success',
            'days_remaining': days_remaining,
            'message': '设备校准状态正常'
        }
```

## 四、过期设备限制使用

### 4.1 限制机制

当设备校准过期后，系统自动实施使用限制：

| 限制项 | 说明 |
|--------|------|
| 禁止新建任务 | 无法使用过期设备创建新的检测任务 |
| 禁止数据录入 | 无法使用过期设备录入检测数据 |
| 禁止报告签发 | 关联过期设备的报告无法签发 |
| 设备状态锁定 | 设备状态自动变更为"停用" |

### 4.2 限制实现

```python
def check_device_available(device_id: int) -> dict:
    """
    检查设备是否可用
    
    Args:
        device_id: 设备ID
    
    Returns:
        dict: 包含可用状态和原因
    """
    device = get_device(device_id)
    current_date = date.today()
    
    # 检查校准状态
    if device.calibration_expiry_date < current_date:
        return {
            'available': False,
            'reason': '设备校准已过期',
            'expiry_date': device.calibration_expiry_date,
            'days_overdue': (current_date - device.calibration_expiry_date).days
        }
    
    # 检查设备状态
    if device.status != 'active':
        return {
            'available': False,
            'reason': f'设备状态为{device.status}，不可使用'
        }
    
    return {
        'available': True,
        'reason': '设备可用'
    }


def validate_device_for_task(device_ids: list) -> tuple:
    """
    验证设备是否可用于任务
    
    Args:
        device_ids: 设备ID列表
    
    Returns:
        tuple: (是否全部可用, 不可用设备列表)
    """
    unavailable_devices = []
    
    for device_id in device_ids:
        result = check_device_available(device_id)
        if not result['available']:
            device = get_device(device_id)
            unavailable_devices.append({
                'device_id': device_id,
                'device_name': device.name,
                'device_code': device.code,
                'reason': result['reason']
            })
    
    return len(unavailable_devices) == 0, unavailable_devices
```

## 九、常见问题

### Q1: 预警阈值可以自定义吗？

**答案**：可以。在系统设置中可以配置预警提前天数和紧急提前天数，不同设备类型也可以设置不同的预警阈值。

### Q2: 设备过期后如何恢复使用？

**答案**：需要完成以下步骤：
1. 安排设备校准
2. 提交校准证书和结果
3. 系统自动更新校准有效期
4. 设备状态恢复为"正常"

### Q3: 如何临时使用过期设备？

**答案**：在特殊情况下，可通过以下流程临时解除限制：
1. 提交临时使用申请
2. 填写风险评估和补偿措施
3. 技术负责人审核
4. 质量负责人批准
5. 临时使用期间系统记录所有操作

### Q4: 预警通知可以关闭吗？

**答案**：可以在系统设置中关闭预警通知，但不建议关闭。关闭通知可能导致设备过期未及时处理。

### Q5: 如何查看设备的历史预警记录？

**答案**：在设备详情页面可以查看设备的所有历史预警记录，包括预警产生时间、处理时间、处理人等信息。

### Q6: 多个设备同时预警如何处理？

**答案**：系统支持批量处理功能：
1. 在预警列表勾选多个设备
2. 点击"批量安排校准"
3. 统一设置校准计划和机构
4. 批量提交处理

### Q7: 校准机构信息在哪里维护？

**答案**：在"系统设置 → 基础数据 → 校准机构"中维护校准机构信息，包括机构名称、联系方式、资质范围等。

### Q8: 设备校准周期如何设置？

**答案**：在设备信息维护中设置校准周期（如12个月），系统会根据上次校准日期自动计算下次校准日期。

## 十、最佳实践

### 10.1 预警管理建议

1. **定期检查预警列表**：每天查看预警列表，及时处理预警
2. **提前安排校准**：收到预警后尽快安排校准，避免紧急情况
3. **建立校准计划**：根据预警情况制定月度/季度校准计划
4. **保持证书完整**：妥善保管校准证书，便于追溯

### 10.2 风险控制措施

1. **备用设备**：关键设备配备备用设备
2. **交叉校准**：同类设备进行交叉校准验证
3. **期间核查**：在两次校准之间进行期间核查
4. **数据追溯**：发现设备异常时追溯历史数据

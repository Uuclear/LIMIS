## 一、委托状态机

### 1.1 状态定义

| 状态 | 代码 | 说明 |
|------|------|------|
| 草稿 | draft | 新建未提交 |
| 待评审 | pending_review | 已提交等待评审 |
| 已评审 | reviewed | 评审通过 |
| 已退回 | rejected | 评审退回 |
| 已取消 | cancelled | 已取消 |

### 1.2 状态流转图

```
                    ┌─────────────────────────────────────┐
                    │                                     │
                    ▼                                     │
┌─────────┐    提交    ┌───────────┐    评审通过    ┌─────────┐
│  草稿   │ ─────────▶ │  待评审   │ ────────────▶ │ 已评审  │
│ (draft) │            │(pending)  │               │(reviewed)│
└─────────┘            └───────────┘               └─────────┘
     │                       │                          
     │ 取消                  │ 评审退回                  
     ▼                       ▼                          
┌─────────┐            ┌─────────┐ ──▶ 修改后重新提交 ──┘
│ 已取消  │            │ 已退回  │

└─────────┘            └─────────┘
```

### 1.3 合法状态转换

| 当前状态 | 可转换状态 | 触发条件 |
|----------|------------|----------|
| draft | pending_review | 提交委托 |
| draft | cancelled | 取消委托 |
| pending_review | reviewed | 评审通过 |
| pending_review | rejected | 评审退回 |
| rejected | draft | 修改后重新提交 |
| reviewed | - | 终态，不可变更 |
| cancelled | - | 终态，不可变更 |

## 六、状态机实现原理

### 6.1 状态定义

每个业务模型中定义合法状态转换：

```python
class Commission(BaseModel):
    VALID_TRANSITIONS = {
        'draft': ['pending_review', 'cancelled'],
        'pending_review': ['reviewed', 'rejected'],
        'rejected': ['draft'],
        'reviewed': [],  # 终态
        'cancelled': [],  # 终态
    }
    
    def can_transition_to(self, new_status: str) -> bool:
        """检查是否可以转换到新状态"""
        return new_status in self.VALID_TRANSITIONS.get(self.status, [])
```

### 6.2 状态变更方法

```python
def change_status(self, new_status: str, user, reason: str = ''):
    """变更状态"""
    if not self.can_transition_to(new_status):
        raise ValidationError(f'不能从 {self.status} 转换到 {new_status}')
    
    old_status = self.status
    self.status = new_status
    self.save()
    
    # 记录状态变更日志
    StatusLog.objects.create(
        obj=self,
        old_status=old_status,
        new_status=new_status,
        changed_by=user,
        reason=reason,
    )
```

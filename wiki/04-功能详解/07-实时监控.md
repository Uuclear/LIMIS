# 实时监控

本文档详细介绍实时监控预警功能的实现和使用方法。

---

## 一、功能概述

实时监控预警功能是LIMS系统的核心监控模块，提供全方位的实验室运行状态监控，及时发现和处理各类预警信息，确保实验室工作顺利进行。

### 1.1 监控范围

| 监控类型 | 说明 |
|----------|------|
| 设备预警 | 设备校准到期、维护保养到期等 |
| 样品预警 | 样品超期、留样到期等 |
| 任务预警 | 任务超期、任务分配异常等 |
| 报告预警 | 报告超期、报告审核异常等 |
| 质量预警 | 质控异常、能力验证结果异常等 |

### 1.2 监控价值

| 价值 | 说明 |
|------|------|
| 实时感知 | 实时掌握实验室运行状态 |
| 及时预警 | 第一时间发现问题并预警 |
| 快速响应 | 快速处理各类异常情况 |
| 数据支撑 | 为管理决策提供数据支撑 |

| 卡片 | 显示内容 | 颜色说明 |

## 四、预警处理流程

### 4.1 预警生命周期

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  预警产生   │───▶│  预警通知   │───▶│  预警处理   │
└─────────────┘    └─────────────┘    └─────────────┘
      │                   │                  │
      ▼                   ▼                  ▼
  系统自动检测        发送通知给          处理预警
  定时任务扫描        相关人员            记录处理结果
                                            │
                                            ▼
                                    ┌─────────────┐
                                    │  预警关闭   │
                                    └─────────────┘
```

### 4.2 预警处理步骤

#### 步骤一：查看预警

1. 进入监控仪表盘或预警列表
2. 查看预警详情
3. 了解预警原因和关联对象

#### 步骤二：分析原因

1. 查看关联对象详情
2. 分析预警产生原因
3. 确定处理方案

#### 步骤三：处理预警

根据预警类型采取相应处理措施：

| 预警类型 | 处理措施 |
|----------|----------|
| 设备校准预警 | 安排设备校准 |
| 样品超期预警 | 加快检测进度 |
| 任务超期预警 | 调整任务分配 |
| 报告超期预警 | 加快审核进度 |
| 质控异常预警 | 分析原因，必要时重新检测 |

#### 步骤四：确认处理

1. 填写处理结果
2. 上传相关附件
3. 确认处理完成

#### 步骤五：预警关闭

系统自动关闭预警或手动关闭预警

### 4.3 预警处理界面

**路径**：监控中心 → 预警列表 → 点击"处理"

**处理表单字段**：

| 字段 | 说明 | 必填 |
|------|------|------|
| 处理方式 | 处理预警的方式 | 是 |
| 处理说明 | 处理过程说明 | 是 |
| 处理结果 | 处理结果描述 | 是 |
| 附件 | 相关证明文件 | 否 |
| 处理人 | 自动获取当前用户 | 自动 |

### 4.4 预警升级机制

当预警未在规定时间内处理时，系统自动升级：

| 升级规则 | 说明 |
|----------|------|
| 预警24小时未处理 | 升级为紧急预警，通知部门负责人 |
| 紧急预警12小时未处理 | 升级为严重预警，通知质量负责人 |
| 严重预警6小时未处理 | 通知实验室主任 |

### 4.5 预警忽略机制

对于非关键预警，可以选择忽略：

**忽略条件**：
- 预警级别为"预警"（非紧急/严重）
- 有合理的忽略理由

**忽略流程**：
1. 点击"忽略"按钮
2. 填写忽略原因
3. 确认忽略

**忽略记录**：
- 系统记录忽略操作
- 忽略原因可追溯
- 忽略后不再重复预警

## 七、API接口

### 7.1 获取预警统计

**接口地址**：`GET /api/monitoring/statistics`

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| start_date | date | 否 | 开始日期 |
| end_date | date | 否 | 结束日期 |
| department_id | int | 否 | 部门ID |

**响应示例**：

```json
{
    "code": 200,
    "message": "success",
    "data": {
        "total": 45,
        "by_type": {
            "device": 12,
            "sample": 8,
            "task": 15,
            "report": 5,
            "quality": 5
        },
        "by_level": {
            "warning": 25,
            "urgent": 15,
            "critical": 5
        },
        "by_status": {
            "pending": 10,
            "processing": 5,
            "completed": 30
        },
        "process_rate": 0.889,
        "avg_process_time": 3.5
    }
}
```

### 7.2 获取预警列表

**接口地址**：`GET /api/monitoring/warnings`

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | string | 否 | 预警类型：device/sample/task/report/quality |
| level | string | 否 | 预警级别：warning/urgent/critical |
| status | string | 否 | 处理状态：pending/processing/completed |
| page | int | 否 | 页码 |
| page_size | int | 否 | 每页数量 |

**响应示例**：

```json
{
    "code": 200,
    "message": "success",
    "data": {
        "total": 45,
        "page": 1,
        "page_size": 20,
```

## 八、常见问题

### Q1: 预警通知太频繁怎么办？

**答案**：可以在系统设置中调整通知频率：
1. 进入"系统设置 → 监控配置 → 通知设置"
2. 调整通知频率和重复通知间隔
3. 可以关闭非紧急预警的通知

### Q2: 如何查看历史预警记录？

**答案**：在预警列表页面：
1. 点击"高级筛选"
2. 选择时间范围
3. 选择"已处理"状态
4. 点击查询即可查看历史预警

### Q3: 预警可以批量处理吗？

**答案**：可以。在预警列表页面：
1. 勾选需要处理的预警
2. 点击"批量处理"按钮
3. 填写处理结果
4. 确认提交

### Q4: 如何自定义预警阈值？

**答案**：在系统设置中可以自定义各类预警的阈值：
1. 进入"系统设置 → 监控配置 → 预警规则"
2. 选择需要修改的预警类型
3. 修改预警阈值和紧急阈值
4. 保存配置

### Q5: 预警升级规则可以修改吗？

**答案**：可以。在系统设置中可以修改升级规则：
1. 进入"系统设置 → 监控配置 → 升级规则"
2. 修改升级时间和升级动作
3. 保存配置

### Q6: 如何关闭某类预警？

**答案**：在预警规则配置中可以关闭某类预警：
1. 进入"系统设置 → 监控配置 → 预警规则"
2. 找到需要关闭的预警类型
3. 关闭开关
4. 保存配置

### Q7: 预警数据可以导出吗？

**答案**：可以。在预警列表或统计页面：
1. 点击"导出"按钮
2. 选择导出格式（Excel/PDF）
3. 选择导出范围
4. 确认导出

### Q8: 如何查看预警处理效率？

**答案**：在统计分析页面可以查看：
1. 进入"监控中心 → 统计分析"
2. 查看预警处理率、及时处理率等指标
3. 可以按时间、部门等维度分析

## 九、最佳实践

### 9.1 日常监控建议

1. **每日查看仪表盘**：每天上班后首先查看监控仪表盘
2. **及时处理预警**：收到预警后尽快处理，避免升级
3. **定期分析趋势**：每周分析预警趋势，找出问题根源
4. **持续优化配置**：根据实际情况调整预警阈值

### 9.2 预警处理建议

1. **优先处理紧急预警**：按预警级别优先处理紧急和严重预警
2. **记录处理过程**：详细记录预警处理过程和结果
3. **分析根本原因**：不只是处理预警，要分析根本原因
4. **制定预防措施**：针对常见预警制定预防措施

### 9.3 配置优化建议

1. **合理设置阈值**：根据实际情况设置合理的预警阈值
2. **精准配置接收人**：确保预警通知发送给正确的人
3. **定期检查配置**：定期检查和优化预警配置
4. **避免预警疲劳**：避免预警过多导致预警疲劳

# 盲样管理

本文档详细介绍盲样管理功能的实现和使用方法。

---

## 一、功能概述

盲样管理是保证检测公正性的重要措施。通过盲样编号，检测人员无法获知样品的真实来源和委托方信息。

### 1.1 盲样概念

- **真实编号**：样品的实际编号，包含委托方信息
- **盲样编号**：隐藏委托方信息的编号，仅用于检测人员

### 1.2 盲样目的

- 保证检测公正性
- 防止利益冲突
- 符合ISO/IEC 17025要求

## 二、盲样编号生成

### 2.1 编号格式

盲样编号格式：`BL-YYYYMM-XXXXXX`

| 部分 | 说明 |
|------|------|
| BL | 盲样前缀 |
| YYYYMM | 年月 |
| XXXXXX | 6位随机字符 |

### 2.2 生成规则

- 自动生成唯一编号
- 编号不包含任何样品信息
- 编号不可预测

### 2.3 生成方法

```python
def generate_blind_no():
    """生成盲样编号"""
    prefix = 'BL'
    date_part = datetime.now().strftime('%Y%m')
    random_part = ''.join(random.choices('ABCDEFGHJKLMNPQRSTUVWXYZ23456789', k=6))
    return f'{prefix}-{date_part}-{random_part}'
```

## 三、盲样分配

### 3.1 单个分配

**路径**：样品管理 → 样品详情 → 分配盲样编号

1. 进入样品详情页
2. 点击「分配盲样编号」按钮
3. 系统自动生成盲样编号
4. 确认分配

### 3.2 批量分配

**路径**：样品管理 → 样品列表 → 批量分配盲样编号

1. 在样品列表勾选多个样品
2. 点击「批量分配盲样编号」
3. 系统为所有选中样品生成盲样编号
4. 确认分配

### 3.3 API接口

```
# 单个分配
POST /api/v1/samples/samples/{id}/assign-blind-no/

# 批量分配
POST /api/v1/samples/samples/batch-assign-blind-no/
参数：
{
  "sample_ids": [1, 2, 3, 4, 5]
}

返回：
{
  "success": true,
  "blind_numbers": {
    "1": "BL-202603-ABC123",
    "2": "BL-202603-DEF456",
    ...
  }
}
```

## 四、盲样映射表

### 4.1 映射表内容

盲样映射表记录盲样编号与真实编号的对应关系：

| 盲样编号 | 真实编号 | 委托单号 | 委托方 |
|----------|----------|----------|--------|
| BL-202603-ABC123 | YP-20260301-001 | WT-20260301-001 | XX建设公司 |

### 4.2 查看权限

只有以下角色可以查看盲样映射表：
- 系统管理员
- 技术负责人
- 质量负责人

- 设备管理员

**检测人员不能查看映射表**

### 4.3 查看路径

**路径**：样品管理 → 盲样映射表

## 五、盲样使用

### 5.1 检测人员视角

检测人员在系统中只能看到：
- 盲样编号
- 样品类型
- 检测项目
- 检测方法

**看不到的信息**：
- 真实样品编号
- 委托方信息
- 工程项目信息
- 样品来源

### 5.2 数据权限控制

系统通过数据权限控制实现盲样：

```python
class SampleViewSet(viewsets.ModelViewSet):
    def get_queryset(self):
        user = self.request.user
        if user.role == 'tester':
            # 检测人员只能看到盲样编号
            return Sample.objects.filter(
                blind_no__isnull=False
            ).only('blind_no', 'sample_type', 'test_items')
        else:
            # 其他角色可以看到完整信息
            return Sample.objects.all()
```

## 六、盲样解除

### 6.1 解除场景

以下情况需要解除盲样：
- 检测完成，需要出具报告
- 样品退还，需要通知委托方
- 异常处理，需要追溯来源

### 6.2 解除权限

只有样品管理员和技术负责人可以解除盲样。

### 6.3 解除操作

```
POST /api/v1/samples/samples/{id}/remove-blind-no/

返回：
{
  "success": true,
  "message": "盲样编号已解除"
}
```

## 八、常见问题

### Q1: 盲样编号可以修改吗？

**答案**：不可以。盲样编号一旦分配不可修改。

### Q2: 检测人员如何知道样品来源？

**答案**：检测人员不应该知道样品来源，这是盲样管理的目的。

### Q3: 盲样编号重复怎么办？

**答案**：系统会检查唯一性，重复时会自动重新生成。

### Q4: 所有样品都需要盲样吗？

**答案**：建议对涉及公正性的检测样品使用盲样管理。

# 电子签名

本文档详细介绍电子签名功能的实现和使用方法。

---

## 一、功能概述

电子签名是用于报告签署的数字化签名，具有法律效力。根据《电子签名法》，可靠的电子签名与手写签名具有同等法律效力。

### 1.1 适用场景

- 检测报告签署
- 原始记录签署
- 审批流程确认

### 1.2 适用角色

- 授权签字人：可签署检测报告
- 技术负责人：可签署批准意见
- 审核人：可签署审核意见

---

## 二、签名上传

### 2.1 签名要求

| 要求项 | 规范 |
|--------|------|
| 文件格式 | PNG、JPG |
| 文件大小 | ≤ 2MB |
| 图片尺寸 | 建议 200×100 像素 |
| 背景 | 透明或白色 |
| 清晰度 | 签名清晰可辨 |

### 2.2 上传步骤

**路径**：个人中心 → 电子签名

1. 点击「上传签名」按钮
2. 选择签名图片文件
3. 系统验证格式和大小
4. 预览签名效果
5. 确认上传

### 2.3 API接口

```
POST /api/v1/system/users/me/signature/
Content-Type: multipart/form-data

参数：
- signature: 签名图片文件

返回：
{
  "success": true,
  "signature_url": "/media/signatures/user_001.png",
  "uploaded_at": "2026-03-31T10:00:00Z"
}
```

## 三、签名验证

### 3.1 签名有效性检查

系统在签署报告时自动检查签名有效性：

| 检查项 | 说明 |
|--------|------|
| 签名是否存在 | 用户是否已上传签名 |
| 签名格式 | 是否为有效图片格式 |
| 用户权限 | 是否有签署权限 |
| 授权范围 | 是否在授权范围内 |

### 3.2 签名信息存储

签名信息存储在用户模型中：

| 字段 | 说明 |
|------|------|
| signature | 签名图片路径 |
| signature_updated_at | 签名上传时间 |
| certificate_no | 资质证书编号 |
| certificate_expiry | 资质证书有效期 |

---

## 四、报告签署

### 4.1 签署流程

1. 授权签字人批准报告
2. 系统自动检查签名有效性
3. 系统将签名添加到报告
4. 记录签署时间和签署人

### 4.2 签名位置

签名添加到报告的指定位置：
- 批准人签字栏
- 签署日期自动记录

### 4.3 签署记录

每次签署都会记录：

| 字段 | 说明 |
|------|------|
| 签署人 | 签署用户 |
| 签署时间 | 签署时间 |
| 签署类型 | 批准/审核 |
| 签名图片 | 签名图片路径 |
| 报告编号 | 关联报告 |

## 五、签名安全

### 5.1 签名保护

- 签名图片存储在安全目录
- 签名文件访问需权限验证
- 签名使用记录可追溯

### 5.2 签名更换

更换签名需：
1. 删除旧签名
2. 上传新签名
3. 新签名立即生效
4. 旧报告签名不变

### 5.3 签名删除

```
DELETE /api/v1/system/users/me/signature/

返回：
{
  "success": true,
  "message": "签名已删除"
}
```

## 六、法律效力

### 6.1 电子签名法要求

可靠的电子签名需满足：
1. 签名制作数据由签名人专有
2. 签名制作数据仅由签名人控制
3. 签名后任何改动能够被发现
4. 签名后数据任何改动能够被发现

### 6.2 系统实现

本系统通过以下方式确保签名可靠性：
- 签名图片由用户自行上传
- 签名使用需用户登录认证
- 签署记录不可篡改
- 报告数据完整性校验

## 七、常见问题

### Q1: 签名上传失败怎么办？

**排查步骤**：
1. 检查文件格式是否为PNG或JPG
2. 检查文件大小是否超过2MB
3. 检查图片是否清晰
4. 尝试更换浏览器

### Q2: 签名可以借给他人使用吗？

**答案**：不可以。签名仅限本人使用，借给他人使用违反规定。

### Q3: 如何更换签名？

**方法**：先删除旧签名，再上传新签名。

### Q4: 签名被盗用怎么办？

**处理**：立即联系系统管理员，删除签名并调查盗用情况。

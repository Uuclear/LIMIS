# 角色权限说明

本文档详细介绍系统的角色体系和权限配置，帮助用户了解自己的操作权限范围。

---

## 一、角色体系概述

系统采用**RBAC（基于角色的访问控制）**模型，通过角色将权限分配给用户。每个用户可以拥有一个或多个角色，其权限为所有角色权限的并集。

### 1.1 角色列表

系统预设了11种角色，覆盖实验室各类人员：

| 角色名称 | 角色代码 | 典型职责 |
|----------|----------|----------|
| 系统管理员 | `admin` | 系统配置、用户管理、权限分配 |
| 技术负责人 | `tech_director` | 技术管理、报告批准、资源调配 |
| 质量负责人 | `quality_director` | 质量管理、内审管评、不符合处理 |
| 授权签字人 | `auth_signer` | 报告签署、技术审核 |
| 审核人 | `reviewer` | 报告审核、记录复核 |
| 监督员 | `supervisor` | 质量监督、现场检查 |
| 检测人员 | `tester` | 检测执行、记录填写 |
| 样品管理员 | `sample_clerk` | 样品登记、流转管理、处置 |
| 设备管理员 | `equip_manager` | 设备管理、校准维护 |
| 接待员 | `reception` | 委托登记、合同评审、报告发放 |
| 委托方 | `client` | 委托提交、进度查询、报告查看 |

## 三、权限粒度说明

### 3.1 模块级权限

控制用户是否可以访问某个功能模块。

```
示例：检测人员可以访问"检测任务"模块，但不能访问"用户管理"模块
```

### 3.2 操作级权限

控制用户在模块内可以执行的操作类型：

| 操作类型 | 代码 | 说明 |
|----------|------|------|
| 查看 | `view` | 查看列表和详情 |
| 创建 | `create` | 新增数据 |
| 编辑 | `edit` | 修改数据 |
| 删除 | `delete` | 删除数据 |
| 审批 | `approve` | 审批流程操作 |
| 导出 | `export` | 导出数据 |

### 3.3 按钮级权限

控制页面内特定按钮的显示和可用性：

```vue
<!-- 示例：只有有编辑权限的用户才能看到编辑按钮 -->
<el-button v-permission="'sample:edit'">编辑</el-button>

<!-- 示例：没有权限时按钮禁用 -->
<el-button v-permission-disabled="'report:approve'">批准</el-button>
```

### 3.4 数据级权限

控制用户可以访问的数据范围：

| 权限范围 | 说明 |
|----------|------|
| 全部数据 | 可以查看所有数据 |
| 本部门数据 | 只能查看本部门的数据 |
| 本人数据 | 只能查看自己创建/负责的数据 |
| 委托方数据 | 只能查看自己提交的委托相关数据 |

## 四、权限配置示例

### 4.1 新增用户并分配角色

1. 进入「系统管理」→「用户管理」
2. 点击「新增」按钮
3. 填写用户基本信息
4. 在「角色」下拉框中选择角色（可多选）
5. 点击「保存」

### 4.2 修改用户角色

1. 进入「系统管理」→「用户管理」
2. 找到目标用户，点击「编辑」
3. 修改角色选择
4. 点击「保存」

### 4.3 查看用户权限

1. 进入「系统管理」→「用户管理」
2. 点击用户详情
3. 查看「权限列表」标签页
4. 显示该用户所有角色的权限并集

## 五、权限常见问题

### Q1: 为什么我看不到某个菜单？

**原因**：您没有访问该模块的权限。

**解决方法**：联系系统管理员分配相应角色。

### Q2: 为什么按钮是灰色的？

**原因**：您没有执行该操作的权限。

**解决方法**：联系系统管理员确认权限配置。

### Q3: 我有多个角色，权限如何计算？

**答案**：多角色权限取并集，即您拥有所有角色的权限总和。

### Q4: 为什么我不能编辑某条数据？

**可能原因**：
1. 没有编辑权限
2. 数据状态不允许编辑（如已审批）
3. 数据属于他人且您没有全部数据权限

### Q5: 如何申请更多权限？

**解决方法**：向系统管理员提交权限申请，说明需要访问的模块和原因。

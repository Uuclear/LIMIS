# 浦东国际机场四期扩建工程 - 实验室信息管理系统 (LIMIS)

## 项目概述

本项目是为上海浦东国际机场四期扩建工程专门开发的第三方检测实验室信息管理系统。系统基于Django + Vue3技术栈，实现了完整的实验室业务流程管理，包括项目管理、委托管理、样品管理、检测业务、报告管理、质量体系等功能模块。

## 系统架构

### 技术栈
- **后端**: Django 5.x + Django REST Framework
- **数据库**: PostgreSQL 16
- **缓存**: Redis 7
- **异步任务**: Celery + Redis
- **对象存储**: MinIO
- **认证**: JWT Token
- **文档**: drf-spectacular (Swagger/OpenAPI)

- **前端**: Vue 3 + TypeScript
- **UI 库**: Element Plus
- **状态管理**: Pinia
- **构建工具**: Vite
- **图表**: ECharts

### 部署方案
- **容器化**: Docker + Docker Compose
- **Web 服务器**: Nginx
- **应用服务器**: Gunicorn

## 机场工程特色功能

### 1. 标准规范管理
- 支持机场工程相关标准规范（GB、JT、MH等）
- 工标网标准自动抓取功能
- 标准状态管理（现行、即将实施、已废止）

### 2. 检测参数库
- 涵盖土木工程、混凝土、钢筋、沥青、土工等领域
- 完整的检测方法和参数体系
- 按机场工程特点分类管理

### 3. 机场工程检测流程
- **委托管理**: 机场工程专用委托单类型
- **样品管理**: 机场工程样品分类和存储管理
- **检测执行**: 专业检测方法和原始记录模板
- **报告管理**: 机场工程检测报告模板和审批流程

### 4. 全程追溯功能
- 项目→委托→样品→检测→报告完整追溯链
- 操作人员责任追溯
- 设备使用追溯
- 环境条件追溯

### 5. 实时监控预警
- 任务进度监控
- 质量指标监控
- 预警阈值设定
- 多级预警机制

### 6. 资质认证合规
- 符合ISO/IEC 17025技术要求
- 满足CNAS认可要求
- 交通工程检测资质要求
- 完整的质量管理体系

## 系统部署

### 环境要求
- **操作系统**: Linux (推荐)、macOS、Windows (WSL2)
- **Docker**: 20.10+
- **Docker Compose**: v2
- **Git**: 2.20+
- **资源建议**: CPU ≥ 2 核，内存 ≥ 4 GB

### 部署步骤
1. 克隆代码库
2. 复制环境配置文件
3. 构建并启动容器
4. 初始化数据
5. 访问系统

### 初始化脚本
运行 `/opt/limis/init_airport_system.sh` 脚本完成系统初始化。
